# 本项目使用的技术总结

## 文档说明

本文档用于整理 CodeView 当前仓库中已经实际使用到的技术、库和工程能力，基于现有代码与依赖文件汇总，不包含未落地的预留方案。

## 项目整体形态

CodeView 是一个前后端分离的 GitHub 数据可视化项目，采用 monorepo 结构。

- 前端目录：`apps/web`
- 后端目录：`apps/server`
- 数据库：本地 `SQLite`
- 外部数据源：`GitHub REST API`

## 核心语言与基础技术

### 1. TypeScript

- 前端和后端都使用 `TypeScript`
- 用于类型约束、接口定义和工程统一
- 构建时通过 `tsc` 做类型检查和编译

### 2. Node.js

- 作为后端运行时
- 承载 API 服务、定时同步、数据库读写和数据聚合

### 3. GitHub REST API

- 项目通过服务端请求 GitHub 官方 REST API
- 用于拉取用户信息、仓库列表、提交记录、语言分布、流量数据和关键文件内容

## 前端技术栈

### 1. React

- 前端核心框架
- 当前项目使用函数组件与 Hooks
- 页面通过路由分模块组织

### 2. Vite

- 前端开发服务器与构建工具
- 用于本地开发、打包构建和静态资源处理

### 3. React Router

- 使用库：`react-router-dom`
- 用于前端页面路由管理
- 当前页面包括首页、项目列表、项目详情、技术栈分析、洞察中心、数据统计、配置中心

### 4. Zustand

- 用于轻量级全局状态管理
- 当前主要管理用户 ID、CSRF Token、配置数据和选中仓库状态

### 5. SCSS

- 使用 `sass` 支持 `SCSS`
- 用于页面样式、变量管理和样式复用
- 当前项目使用独立页面样式文件和公共样式目录

### 6. ECharts

- 使用库：`echarts`、`echarts-for-react`
- 用于绘制图表和数据可视化内容
- 当前用于趋势图、热力图、统计图和技术栈相关图形展示

### 7. Lucide React

- 使用库：`lucide-react`
- 用于界面图标展示
- 当前主要用于侧边栏导航和局部状态表达

## 后端技术栈

### 1. Express

- 后端 Web 框架
- 用于提供 `/api` 接口
- 当前承载配置、同步、看板、仓库、技术栈等模块接口

### 2. Zod

- 用于环境变量校验和接口参数校验
- 当前用于请求入参约束、配置结构校验和运行时兜底验证

### 3. CORS

- 使用库：`cors`
- 用于处理前后端跨域访问控制
- 当前根据 `WEB_ORIGIN` 生成允许访问来源

### 4. date-fns / date-fns-tz

- 用于日期计算、统计窗口处理和时区转换
- 当前用于近 7 天、30 天、90 天指标统计，以及日报、周报、月报桶聚合

### 5. crypto

- 使用 Node.js 内置 `crypto`
- 用于生成 CSRF Token 和加密保存 GitHub Token

## 数据库与数据存储

### 1. SQLite

- 项目当前使用 `SQLite` 作为本地数据库
- 适合单机部署、个人项目和轻量化数据持久化

### 2. better-sqlite3

- SQLite 的 Node.js 驱动
- 用于数据库连接、参数化查询、建表、迁移和聚合数据处理
- 当前数据库文件路径由 `DATABASE_PATH` 控制

## 数据同步与调度

### 1. GitHub 数据同步

- 后端会把 GitHub 数据拉取到本地数据库
- 同步内容包括：
  - 用户仓库列表
  - 提交记录
  - 语言分布
  - 流量数据
  - 仓库关键文件快照

### 2. 定时同步

- 当前项目实现了定时增量同步机制
- 服务启动时会恢复已保存的同步计划
- 默认同步周期是 `720` 分钟，即 `12` 小时

说明：

- 代码里安装了 `node-cron`，但当前定时调度实际使用的是 `setTimeout` 方案
- 因此可以认为项目具备“定时任务能力”，但并未真正使用 `node-cron` 作为当前执行器

## 工程化与开发工具

### 1. npm workspaces

- 根目录使用 `workspaces` 管理 `apps/server` 和 `apps/web`
- 方便统一安装依赖和执行脚本

### 2. concurrently

- 用于本地开发时同时启动前端和后端服务
- 对应根脚本：`npm run dev`

### 3. tsx

- 用于后端开发环境直接运行 TypeScript
- 当前服务端开发命令采用 `tsx watch`

### 4. tsc-alias

- 用于构建后处理 TypeScript 路径别名
- 保证服务端编译产物中的路径可正确解析

### 5. tsconfig-paths

- 用于开发态支持 TypeScript 路径别名
- 当前服务端开发启动时通过 `-r tsconfig-paths/register` 注入

## 当前实际技术清单

按“已在仓库中实际使用”汇总如下：

- 语言：`TypeScript`
- 前端框架：`React`
- 前端构建：`Vite`
- 路由：`React Router`
- 状态管理：`Zustand`
- 样式：`SCSS`
- 图表：`ECharts`
- 图标：`Lucide React`
- 后端框架：`Express`
- 参数校验：`Zod`
- 数据库：`SQLite`
- SQLite 驱动：`better-sqlite3`
- 跨域处理：`cors`
- 时间处理：`date-fns`、`date-fns-tz`
- 运行时：`Node.js`
- 同步数据源：`GitHub REST API`
- 本地开发并行执行：`concurrently`
- 服务端开发运行：`tsx`
- 路径别名处理：`tsconfig-paths`、`tsc-alias`

## 补充说明

- 项目目前没有接入容器编排
- 项目目前没有使用 Redis
- 项目目前没有接入 MySQL / PostgreSQL
- 项目目前没有使用消息队列
- 项目目前没有使用 SSR
- 项目目前没有使用 GitHub GraphQL API
