# AGENTS.md

## 项目说明

- 项目中文名：CodeView
- 项目描述：一个基于 GitHub 数据的可视化产品，展示项目活跃度、提交趋势、技术栈画像和基础经营数据。
- 项目英文名：`CodeView`
- 项目类型：个人 GitHub 数据可视化产品
- 核心目标：展示项目活跃度、提交趋势、技术栈画像和基础经营数据

## 技术栈要求

- 前端必须使用 `React + Vite + TypeScript`
- 图表使用 `ECharts`
- 后端使用 `Node.js + TypeScript`
- 数据库使用 `SQLite`
- 样式必须使用 `SCSS`

## 开发强约束

- 必须使用 TypeScript，禁止使用 `any`
- 所有函数、变量、接口响应必须定义类型
- 组件必须使用函数组件和 Hooks，禁止 class component
- 注释必须使用中文
- 禁止使用 `eslint-disable`
- 必须移除未使用的变量、函数、import、样式
- 不允许修改无关文件

## 接口与错误处理

- 所有请求必须包含 `userId`
- 所有请求必须处理 `loading` 和 `error`
- 不允许假设接口返回结构
- 成功响应统一为：
  - `success: true`
  - `code: 200`
  - `message: 操作成功`
  - `data: {}`
- 失败响应统一为：
  - `success: false`
  - `code: ERROR_CODE`
  - `message: 错误描述（中文）`
  - `data: null`

## 样式规范

- 必须使用 `SCSS`
- 禁止污染全局样式
- className 必须语义化
- 命名使用统一规范，推荐 BEM
- 样式文件必须包含中文注释，至少说明：
  - 文件说明
  - 样式说明
  - 关键布局说明

## 目录规范

- 公共组件放在 `src/components/commons/组件名/`
- 每个公共组件目录必须包含：
  - `index.tsx`
  - `index.scss`
- 页面目录放在 `src/pages/页面名/`
- 页面文件使用：
  - `index.tsx`
  - `index.scss`
- 工具函数放在 `src/utils/`
- 类型定义放在 `src/types/`
- 公共样式放在 `src/styles/`

## 组件规范

- 每个组件文件必须包含头部中文注释
- Props 或接口定义必须写清楚：
  - 类型
  - 含义
  - 是否必填
  - 默认值

## 工具函数与 API 规范

- 所有工具函数和 API 封装函数必须添加中文注释
- 请求必须集中封装，不允许页面内直接散写请求逻辑
- 不允许编造接口
- 不明确的接口字段必须先确认

## 数据库与安全规范

- 数据库查询必须使用参数化查询
- 禁止拼接 SQL
- 数据库相关注释必须使用中文
- 禁止硬编码 Token、密码、密钥，必须使用环境变量
- POST 请求必须包含 CSRF 令牌，并在服务端校验
- 所有用户输入必须做 HTML 转义，防止 XSS

## 状态与性能规范

- 小型状态使用 `useState`
- 跨组件状态使用 `Zustand`
- 禁止滥用全局状态
- 列表必须使用稳定 `key`
- 大组件必须拆分
- 使用 `useMemo` / `useCallback` 做必要优化
- 路由组件支持懒加载

## 日志规范

- 错误必须记录日志
- API 失败必须记录错误信息
- 禁止输出敏感信息
- 开发环境允许 `console`
- 生产环境必须移除无关日志

## 工作流要求

### 新功能开发

1. 分析需求
2. 确认接口
3. 定义 TypeScript 类型
4. 编写组件结构
5. 编写 SCSS
6. 接入 API
7. 补全错误处理
8. 自检是否符合本文件

### 修改代码

1. 先阅读原代码
2. 理解业务逻辑
3. 明确修改点
4. 再开始修改

### Debug

1. 分析报错
2. 定位问题
3. 找到根因
4. 给出修复方案

## AI 行为约束

- 不允许编造接口
- 不允许假设数据库结构
- 不允许跳过鉴权
- 不允许省略错误处理
- 不允许生成未使用代码
- 不明确需求必须先询问

## 输出前自检

- 是否全部使用 TypeScript 且无 `any`
- 是否所有请求都包含 `userId`
- 是否补全错误处理
- 是否使用 `SCSS`
- 是否符合目录结构
- 是否有未使用代码
- 是否包含中文注释

## 视觉要求

- 保持个性化风格
- 不使用烂大街的蓝紫霓虹科技风
- 优先保证信息层级、可读性和统一性
